# iOS architecture reference

Structural reference for the Kairō iOS codebase under `ios/`. The Xcode project is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen) from `ios/project.yml`; the `.xcodeproj` is derived output and is regenerated (`xcodegen generate`) after any `project.yml` change. For task-oriented guides see [How to sideload Kairō onto an iPhone](../how-to/sideload-iphone.md) and [How to regenerate App Store screenshots](../how-to/regenerate-screenshots.md).

## Project facts

| Property | Value |
|---|---|

## Layer map

| Directory | Contents |
|---|---|
| `Kairo/App/` | `KairoApp` (`@main`), `RootView` (5-tab `TabView`: Home, Capture, Ask, Review, Digest), `AppEnvironment` — the DI container with two wirings: `live()` (FastAPI backend + `LocalCache`) and `standalone()` (fully on-device), selected by `AppConfig.standalone` |
| `Kairo/Config/` | `AppConfig` (see table below) |
| `Kairo/Features/` | `Home`, `Capture`, `Ask`, `Review`, `Digest`, `Settings` — each a SwiftUI View plus a `@MainActor @Observable` ViewModel that depends only on repository/service protocols |
| `Kairo/Core/Models/` | `Memory`, `Card`, `CaptureSummary`, `RAGResponse`, `Stats`, `Proactive`, `ThemeMode` |
| `Kairo/Core/Services/` | `GenerationService` (Apple Foundation Models, iOS 26+ / Apple Intelligence; `isAvailable == false` elsewhere), `CloudGenerationService` (POSTs the already-built prompt to `AppConfig.cloudGenerationURL`; respects the `cloudAnswersEnabled` user opt-out, default on), `EmbeddingService` (`NLEmbedding` sentence embeddings + cosine similarity) with `TextChunker` (sentence-ish chunk splitting), `DomainClassifier` (keyword-based 7-domain tagging), `TranscriptionService` protocol with `AppleSpeechTranscription`, `WhisperKitTranscription`, `RemoteTranscription`, `UnavailableTranscription`, `NotificationService` (local evening streak reminders via `UNUserNotificationCenter`), `AudioRecording` protocol + `AVAudioRecording`, `KairoAPI` protocol + `KairoAPIClient`, `OnDeviceSearch` |
| `Kairo/Core/Repositories/` | Protocols `MemoryRepository`, `CardRepository`, `ProactiveRepository`; `Local*` implementations (backed by `OnDeviceStore`, used in standalone) and `Default*` implementations (backed by `KairoAPI` + `LocalCache`, used in backend mode) |
| `Kairo/Core/Storage/` | `OnDeviceStore` (single-JSON-file store, schema below), `LocalCache` (JSON file cache in Caches for backend-mode offline reads), `DemoData` (idempotent first-run seed with on-device embeddings) |
| `Kairo/Core/Utils/` | `SM2` (spaced-repetition scheduler; ratings again→2, hard→3, good→4, easy→5, quality < 3 lapses), `StreakCalc` (`Streaks.swift`), `Theme`, extensions (`Color+Hex`, `Date+Pretty`) |
| `Kairo/Core/Views/Reusable/` | `DomainTag`, `GoldButton`, `MemoryRow` |
| `KairoShared/` | Compiled into both app and widget: `KairoSun` (`SunMood` + sun face view), `StreakSnapshot` + `SharedStore` (App Group `UserDefaults` bridge, key `streak_snapshot`), `StreakWidgetCard` (`StreakWidgetContent`, `WidgetPalette`) |

### Transcription chains

| Mode | Chain (first available wins) |
|---|---|
| Standalone (`standalone == true`) | `AppleSpeechTranscription` → `WhisperKitTranscription` → `UnavailableTranscription` (clear error; no server exists) |
| Backend, `useOnDeviceTranscription == true` | `WhisperKitTranscription` → `RemoteTranscription` (`/api/transcribe`) |
| Backend, `useOnDeviceTranscription == false` | `RemoteTranscription` only |

## `AppConfig` keys (`ios/Kairo/Config/AppConfig.swift`)

| Key | Type | Committed value | Meaning |
|---|---|---|---|
| `baseURL` | `URL` | `http://localhost:8000` | FastAPI backend address; only used when `standalone == false` |
| `useOnDeviceTranscription` | `Bool` | `true` | Prefer WhisperKit over the backend transcription endpoint (backend mode only) |
| `standalone` | `Bool` | `true` | Selects the fully on-device wiring (`AppEnvironment.standalone()`); `false` selects `live()` |
| `apiToken` | `String?` | `nil` | Bearer token for a secured deployed backend (`KAIRO_API_TOKEN`); `nil` sends no auth header |
| `cloudGenerationURL` | `URL?` | `nil` | HTTPS stateless cloud-generation proxy, used only when Apple Foundation Models are unavailable on the device; `nil` stays fully on-device (extractive answers) |
| `cloudGenerationToken` | `String?` | `nil` | Shared bearer token matching the proxy's `SHARED_TOKEN` |

Convention: `cloudGenerationURL` and `cloudGenerationToken` are deliberately `nil` in the committed file. Real values (a deployed proxy URL such as `<your-proxy-url>` and its `<token>`) live only as a local, never-committed diff on a developer's machine. The proxy receives only the assembled prompt (top-k retrieved snippets + the question); raw memories and the LLM API key never leave their respective sides.

## `kairo-store.json` schema (`OnDeviceStore`)

One JSON file in the app's Documents directory. Decoding tolerates missing keys (each field falls back to its empty default), so older stores and future field additions load without data loss. A file that fails to decode is moved aside to `kairo-store.corrupt.json` before the store continues with an empty snapshot, preserving the bytes for recovery.

### Top level (`Snapshot`)

| Field | Type | Description |
|---|---|---|
| `memories` | `[StoredMemory]` | Captured entries with embeddings |
| `cards` | `[StoredCard]` | Review cards with SM-2 state |
| `reviewDates` | `[String]` | Dates a review was completed |
| `checkInDates` | `[String]` | Explicit check-in dates (in addition to capture days) |
| `prefs` | `[String: String]` | Small key/value preferences (see below) |

### `StoredMemory`

| Field | Type | Description |
|---|---|---|
| `chunkId` | `String` | Memory id |
| `text` | `String` | Entry text |
| `domains` | `[String]` | Domain tags |
| `timestamp` | `String` | ISO-8601 UTC |
| `sourceType` | `String` | Capture source |
| `embedding` | `[Double]` | Whole-entry embedding (fallback + short entries) |
| `chunks` | `[{text, embedding}]?` | Finer-grained retrieval chunks for long entries; `nil` → use `embedding` |

### `StoredCard`

| Field | Type | Description |
|---|---|---|
| `cardId` | `String` | Card id |
| `type` | `String` | `insight` \| `decision` \| `pinned` |
| `front` / `back` | `String` | Card faces |
| `domain` | `String` | Domain tag |
| `createdAt` / `dueDate` | `String` | ISO-8601; `dueDate` drives scheduling |
| `ease` / `intervalDays` | `Double` | SM-2 state |
| `repetitions` / `lapses` | `Int` | SM-2 state |
| `lastReviewed` | `String?` | Last review timestamp |
| `sourceMemoryId` | `String?` | Memory the card was distilled from |

### Notable `prefs` keys

| Key | Value |
|---|---|
| `card_attempted` | Newline-joined memory ids already attempted for card distillation (transient failures are unmarked and retried) |
| `recall:<YYYY-MM-DD>` | JSON-encoded recall card cached for that day |
| `recall_surfaced` | JSON array of memory ids already surfaced as recall cards |
| `recall_response:<memoryId>` | `answered` or `dismissed` |
| `digest:<weekStart>` | JSON-encoded cached weekly digest |

## Build and test commands

Canonical commands, run from `ios/`. The `xattr -cr` step strips extended attributes that Desktop/iCloud checkouts add, which otherwise break ad-hoc codesigning; the Simulator itself needs no signing (`CODE_SIGNING_ALLOWED=NO`).

```bash
xcodegen generate                                # after editing project.yml

xattr -cr Kairo Kairo.xcodeproj                  # Desktop checkouts only
xcodebuild -project Kairo.xcodeproj -scheme Kairo \
  -destination 'platform=iOS Simulator,name=iPhone 17' \
  -derivedDataPath build CODE_SIGNING_ALLOWED=NO build

xcodebuild test -project Kairo.xcodeproj -scheme Kairo \
  -destination 'platform=iOS Simulator,name=iPhone 17' CODE_SIGNING_ALLOWED=NO
```

The 28 unit tests live in `KairoTests/` (`HomeViewModelTests`, `OnDeviceEngineTests`, `AskAndCaptureTests`, `ReviewViewModelTests`, `OnDeviceSearchTests`, with shared doubles in `Mocks.swift`). The screenshot harness runs only under the `KairoScreenshots` scheme — see [How to regenerate App Store screenshots](../how-to/regenerate-screenshots.md).
